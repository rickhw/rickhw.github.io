關於 Complete Think

整理關於這個 Blog 的初衷與資訊，關於作者介紹請點 這裡。

標題

Blog 的標題命名緣由 (先不管文法, 詞性的問題):

  • Complete: 指的是『完整性』，也可以說是 視野、全局
    • 源自於電影 征服情海 的經典台詞：You complete me
    • 加入主詞、受詞後，產生連結 (Connect)
    • 加入時間、空間，會看到有不同的全局
    • 完整 不表示都會，而是表示有了解全盤基礎概念、了解其本質性，之後的創意才有意義
    • 完整代表目標，持續往完整走。
  • Think:
    • 源自於 Thinking in Java 這本書的標題
    • 代表『獨立思考』、『我思，故我在』、 本質、抽象、想像力。
    • 喜歡 IBM ThinkPad 的概念與精神，雖然早已不復在
    • Apple 1997 的 slogan Think Different

思考人、事、時、地、物 … 的過程與本質，連結他們。

描述

先生不知來自何方，亦不知歸去何處，年過而立，參悟生與死，淡泊名與利，但憂天下蒼生。蹤跡走紅塵，藏身山林田野，撫琴於搖滾，振筆於網路，傳道於教學；神遊金庸武俠，往返程式思考，常以此樂而忘眠。

這段話描述我個人的生命歷程、使命、生活、理想、想法。

Banner

Blog 的 Banner 圖檔取自電影 社群網站 (The Social Network)。其實我沒有看過這部電影，但這張圖符合 Blog 的核心思想：Thinking 的具象形式，在玻璃上用數學方程式思考。

另外我喜歡 棕色系 的配色基調，代表 有故事 的概念，這張圖的配色剛好滿足我想要的。

讀者

這個 Blog 的讀者設定是我自己，主要是我個人的學習筆記、心得、想法 … 等。因為想法與記憶很珍貴，但 人的生理 本身是不可靠的，所以寫下來回饋給自己是最好的方法。我常常會閱讀自己整理的東西，然後反覆整理分類、重構、組織。

取之於社會，回饋於社會，如果這想法、筆記對你有用處，歡迎保留作者資訊轉帖。

初衷

我一直都有寫 Blog 的習慣，不同主題放在不同地方而已，像是音樂放在 Blogger、技術曾經放在 pixnet、生活曾經放在 wordpress …

寫東西純粹是記錄學習歷程、分享心得，所以也不太會花時間去包裝、或者行銷，如果有也都是學習角度出發，例如想知道某個 css 的用法、或者 GA 怎麼玩。

Complete Think 是從 2015/01 開始寫的，當時有兩個目的：

  1. 想找個可以用 markdown 寫的平台，但是沒找到適合的，後來找到了 Hexo
  2. 當時正準備 離開前公司，那時候給 SQA Team 禮物就是：教大家如何利用 Blog 學習
    • 學習法則 提到 Blog 概念，就是當時送給他們的，給魚吃不如給釣竿。

Blog 的副標題以這三個字為核心： Thinking、Coding、Learning，這三個分別有以下意義：

  • Thinking：獨立思考、想像力、抽象能力、判斷、哲學
  • Coding：工程技術、實踐方法、科學、STEM
  • Learning：終身學習、活到老學到老、學習態度

類比

我經常會拿 軟體 類比 音樂、武俠、烹飪、語言、科幻 類比，另一個 Blog 喝咖啡聊音樂 有很多類似的想法與心得。像是：

  • 簡單、複雜
  • 吉他練習的實踐原則：專心、彈好、彈滿
  • 正確性與速度
  • 曲式、歷史背景、框架模式
  • 概述『和聲分析』演算法
  • 『怎麼彈』和『彈什麼』

FAQ

每件事情背後都有動機，動機就是初衷，為什麼要做這件事情。老樣子，用 Q and A 方式了解：

Q: 為啥 Blog 標題叫做 Complete Think？

參見段落：標題 說明。

Q: 這些文章都是誰寫的？

都是 Rick Hwang 一個人一字一句敲出來的。

Q: 你為啥要寫這些文章？為名？為利？

我常常會有靈感從腦袋閃過，不管是搭車、跑步、洗澡、睡覺，靈感跟想法是很珍貴的。人的記憶不可靠，寫了十幾年 Blog (Since 2006)、和廿幾年的日記，這些經驗告訴我：寫的過程就是在思考、組織、學習，寫文章給自己讀，Feedback 給自己是很棒的！對個人來講，就是種持續對學習方式。

有天如果我不在了，可以留下一些東西，而這些東西或許對社會會有所幫助。至於名與利，如 Blog 的副標說明一樣：那些都只是附加的，不是最終目的。

Q: 可否轉貼 Blog 的文章？引用圖、文？

可以，請註明出處即可。

Q: 這個 Blog 是用啥寫的？放哪？

文章都是用 markdown 寫的，然後透過 Hexo 產生，網站放在 github，Hexo 相關筆記參閱 Blog 這個 Category。

Q: 文章你都會維護？

會的，我會固定 Review 自己的文章，固定重構。如果發現別字、或者想法上的誤解，歡迎指正。

Q: 文章一直都會保留不動？

我會定期重整索引，目的是為了讓學習重新組織化，組織過程中，如果文章是有價值的，就會留下來，沒價值的就會刪去。

Q: 文章是否有打算出版成書？

如果有機會，不排除。置頂文每篇應該就是一本書的目錄，目前相關置頂有：DevOps 與軟體工程實踐、AWS 應用、經營與管理
