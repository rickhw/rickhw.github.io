真實世界

昨天跟同事聊到識人、用人，我提出這樣的想法，要知道一個人能否解決問題的前提，這個人要意會到什麼『是真實的世界』，舉例個誇張點的例子，例如：

有些人是活在被保護的環境之下長大的，從來就不知道：坐車是要錢的、漢堡不是從樹上長出來的、飛機是人造的，牛肉不見得是牛做的 …

這例子聽起來很誇張？是嗎？

真實世界

日常生活

幾個日常生活要面對的問題：

  • 你知道水電自己家裡一個月，水、電分別要花多少錢？
  • 你知道家裡哪個設備最吃電？怎麼算？
  • 你算過貸款 1000 萬台幣，每個月繳 3 萬，要繳費多久？
  • 你知道，月薪五萬，扣掉生活支出，存 100 萬要花多久時間？
  • 你知道台北市平均房價 1500 萬，要花一個上班族多少時間？
  • 你知道一台 50 萬的車，可以開多久？
  • 你知道台灣的水、電便宜到不可思議？
  • 為什麼：美國人老闆：台灣物價好怪，該低不低該
  • … (可以再寫一百點)

很多看起來很小的事情，其實都是每個人都要面對的。

這是真實的世界，了解真實世界，面對它。

維運就是真實世界

我的 Blog 有很多文章在探討這問題：啥是維運？

  • 再談啥是維運
  • What is Ops?
  • Ops as Code using Serverless
  • 系統維運的精神

這些文章大概有本文標題的特質：面對真實的世界

用一句話來講：維運就是 上線 (Go Live) 後的事情

面試時，我常問底下的問題：

  • 你的系統花多久時間開發？
  • 你們的系統上線前做什麼準備？
  • 怎麼上線的？誰做？
  • 系統部署 / 更新有什麼方法？各個優劣？
  • 上線後有多少流量？尖峰？離峰？
  • 系統看哪些資訊知道是否監康？
  • 遭遇過什麼映像深刻的問題？你怎麼處理？
  • 你知道這系統建置花了多少成本嗎？
  • 你的系統平常如何維護？有停機嗎？
  • 系統有狀況時，你們的處理流程是什麼？

結論

在軟體開發過程中，從以下現象大概可以知道，很多老闆、主管沒面對過真實的世界：

  • 以為導入 自動化 就沒事了。
  • 以為導入 敏捷開發 就可以無限壓榨了
  • 以為買了 XXX 軟體公司就一帆順風
  • 以為請了 XXX 大神，公司就 ….

物理能量不滅定律來講，事情總是要做，是人做，還是機器做。人做人要有水準才做的來，機器在 AI 成熟之前，只是人類的代理人。

人生與社會 真實的樣貌 - 起跑點
